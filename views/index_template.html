Why this is a bad visual

This visual shows quotes and trades across all exchanges for shares of The New York Times Company (NYSE: NYT) for a 50 millisecond period. The graphic is taken a bit out of context (this page contains additional commentary) but the point of the visual is to demonstrate that a great amount of bidding/trading occurs in New York (where the exchanges are located) in the amount of time that is takes (according to the speed of light) for information to reach Los Angeles. This is interesting, however the visual misses the opportunity to help the viewer visualize this geographically.

Dataset

The visual displayed above uses data from the traversal of my Github social network (friends and followers). A node is created for each friend or follower. For each user node, additional nodes are created for each starred repository, with an edge from the repository node to the user node of the user who starred that repository. Repository nodes with more than one edge show that multiple users have starred the same repository. The Alternate Visual linked to above uses similar data, but rather than a traversal of my Github social network, relies on data from the top Github contributors.

About Visual

This visual was creating using D3 Javascript library. It uses a force directed layout to position each node. The forces being simulated are a repulsive charge (all nodes) and an attractive force between connected nodes. This results in nodes with similar edges being grouped together on the graph and nodes with dissimilar connections being pused away to the edges of the visual. reated with D3 force directed layout. Additional information about each node is available by clicking on the node.

Analysis

This visual shows Ego-like graphs with few nodes sharing multiple edges.

About Visual #3

Dataset

This visual uses the same data as shown in Visual #2, however instead of plotting a node for each repository a node is plotted for the language in which the starred repository is written. Nodes with multiple edges therefore show multiple users who have a starred one or more repositories in that language.

Analysis

An interesting characteristic that appears in both visuals presented above is the apparent distinction of two classes of languages. In the center of the graph are common language nodes (Java, C, Python, Javascript, etc) that more users appear to be interested in. On the outer regions of the graph are more esoteric languages (LiveScript, F#, TypeScript, OCaml, etc) that are used by one or only a few users. This is Appear to be two classes of languages: common language with edges to many user nodes, and many uncommon langues with few edges.

Power Law Analysis

A power law function expresses the value of a variable in terms of the power (exponent) of another variable. In this case we explore the power law relationship of the distribution of degree of the nodes in our graph. In other words, we look for evidence that the frequency of the number of edges per node follows a power law.
